# Card Guidelines

A card is a single swipeable screen within a story. It is the atomic unit of the learning experience — one idea, one moment, one step forward in the narrative.

---

## Card Anatomy

Every card has up to four elements:

| Element      | Role                                              | Required? |
| ------------ | ------------------------------------------------- | --------- |
| **Visual**   | Dominant image or graphic - carries the mood      | Yes       |
| **Headline** | Core message in one line - what the student keeps | Yes       |
| **Body**     | 1-3 sentences of supporting context               | Optional  |
| **Prompt**   | A question or micro-action inviting engagement    | Optional  |

If a card needs more than these four elements, it is carrying too much — split it.

---

## Card Types

Different moments in a story call for different card types:

### Hook Card

Opens the story. Creates immediate curiosity or recognition.

- Lead with a striking visual or a bold question
- Headline is provocative, not informative
- Body is optional — sometimes the image and headline are enough
- Example headline: *"You're about to make a decision that could cost €200k. What do you actually know?"*

### Situation Card

Establishes the scenario, character, or context the student will follow.

- Introduce who, where, and what is at stake
- Keep it concrete — a name, a role, a specific moment
- Visual should ground the setting

### Tension Card

Introduces the problem, conflict, or gap in knowledge.

- This is where the student starts to feel the question
- Avoid giving the answer — just make the tension clear
- Prompt is effective here: *"What would you do?"* / *"What's missing here?"*

### Concept Card

Introduces a key idea — briefly, through the lens of the story.

- Name the concept, but anchor it in the scenario
- One concept per card, maximum
- Body can include a short definition or contrast
- Never list multiple concepts on a single card

### Data Card

Presents a statistic, finding, or fact that adds weight to the story.

- Let the number be the headline — make it land visually
- Body provides brief context: source, timeframe, what it means
- Keep it to one data point per card

### Reflection Card

Invites the student to pause and connect.

- Used near the end of a story
- A single open question with no right answer
- No body text — the question should stand alone
- Example: *"When did you last see this in real life?"*

### Cliffhanger Card

The final card. Closes the story without resolving it.

- Ends on the central lesson question
- Creates the feeling of an open loop the student will want to close in class
- Visual should feel forward-looking or unresolved
- Example: *"So — is Porter's Five Forces still useful in a world of platforms? We'll find out."*

## Writing Principles

**Headlines are everything.**
A student who only reads the headline should still take something away. Write the headline first. If it doesn't work on its own, rewrite it before touching the body.

**One idea per card.**
Not one topic — one idea. The difference: a topic is "pricing strategy"; an idea is "price anchoring makes €99 feel cheaper than €100, even when we know the trick." Lead with the idea, not the category.

**Body text is a privilege, not a default.**
Add body text only when the headline cannot carry the meaning alone. When in doubt, cut it. Short body = 1–3 sentences max. No paragraphs.

**Prompts invite, not test.**
A prompt on a card is not a quiz question. It should feel like a conversation opener: *"Sound familiar?"*, *"What would you change?"*, *"Have you seen this?"* Low stakes, high invitation.

**Write for a 10-second read.**
If reading the card takes longer than 10–15 seconds, it has too much text. Cut until it doesn't.

---

## Visual Direction

Each card should be conceived with a visual in mind. When writing a card, note what the visual should convey:

- **Mood:** tense, curious, familiar, surprising
- **Subject:** person, place, object, data visualisation, abstract graphic
- **Composition:** simple and bold — one focal point
- Avoid stock-photo clichés (handshakes, lightbulbs, generic offices)
- Prefer: real-world settings, relatable characters, clean data visuals

## Tone per Card Type

| Card type    | Tone                           |
| ------------ | ------------------------------ |
| Hook         | Bold, provocative, urgent      |
| Situation    | Grounded, clear, scene-setting |
| Tension      | Unsettled, questioning, open   |
| Concept      | Precise, confident, anchored   |
| Data         | Surprising, factual, visual    |
| Reflection   | Calm, open, personal           |
| Cliffhanger  | Forward-looking, unresolved    |

## Student Actions

After reading each card, the student makes one of three decisions:

| Action         | What it means                                              |
| -------------- | ---------------------------------------------------------- |
| **Cheatsheet** | "This is key knowledge I want to keep"                     |
| **Know more**  | "This is interesting — I want to go deeper"                |
| **Park**       | "I don't fully get this yet — I'll come back to it"        |

These are the student's decisions to make — not something the card prescribes. The card does not guide the student toward a particular action. Instead, write every card so that all three decisions remain genuinely available:

- **Cheatsheet is possible** when the headline is self-contained and precise enough to stand alone as a note.
- **Know more is possible** when the card opens a question it does not fully answer.
- **Park is possible** when the card introduces complexity without over-explaining it.

A card that forces a single obvious response — or that tries to close every question — removes meaningful choice from the student.

---

## Quality Checklist

Before finalising a card:

- [ ] Headline works on its own — a student could screenshot just this and still get the idea
- [ ] One idea only — no concept stacking
- [ ] Body is 3 sentences or fewer (or absent)
- [ ] Visual direction is specified
- [ ] Headline is precise enough to be cheatsheet-worthy on its own
- [ ] Card opens at least one question it does not fully answer
- [ ] Fits its role in the story arc (hook / build / cliffhanger)
- [ ] Readable in 10–15 seconds
- [ ] Tone matches the moment in the story
